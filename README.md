# Product CRUD API

REST API for managing a product catalog, built as a technical test with Java 17, Spring Boot 3, MySQL and Flyway.

## Tech stack

- Java 17
- Spring Boot 3
- Maven Wrapper
- MySQL 8
- Flyway
- Spring Data JPA
- Bean Validation
- JUnit 5
- Mockito
- Docker Compose
- Springdoc OpenAPI

## Requirements

- Java 17
- Docker and Docker Compose
- Git

Maven does not need to be installed because the Maven wrapper is included.

## Getting started

Clone the repository:

```bash
git clone https://github.com/SerPS92/product-crud-api.git
cd product-crud-api
```

Start MySQL with Docker Compose:

```bash
docker compose up -d
```

Run the application on Windows:

```powershell
.\mvnw.cmd spring-boot:run
```

Run the application on Linux/Mac:

```bash
./mvnw spring-boot:run
```

API base URL:

```text
http://localhost:8080
```

## Database

- Docker Compose starts a MySQL 8 container.
- Database name: `product_crud_db`
- User: `product_user`
- Password: `product_password`
- Port: `3306`
- Flyway automatically runs migrations from `src/main/resources/db/migration`.
- Hibernate uses `ddl-auto: validate`, so tables are not created or updated by Hibernate.

To reset the database from scratch:

```bash
docker compose down -v
docker compose up -d
```

## Endpoints

| Method | Path | Description |
|---|---|---|
| POST | /api/products | Create a product |
| GET | /api/products?page=0&size=20 | List products with pagination |
| GET | /api/products/{id} | Get product by id |
| PUT | /api/products/{id} | Update product |
| DELETE | /api/products/{id} | Delete product |

## Product fields

| Field | Type | Required | Notes |
|---|---|---|---|
| id | Long | Auto-generated | Generated by database |
| name | String | Yes | Max 150 characters |
| description | String | No | Optional |
| price | BigDecimal | Yes | Must be greater than 0 |
| stock | Integer | Yes | Must be zero or greater |
| createdAt | DateTime | Auto-generated | Set on creation |
| updatedAt | DateTime | Auto-generated | Updated on modification |

## Example requests

Create product:

```bash
curl -X POST http://localhost:8080/api/products \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Laptop Lenovo\",\"description\":\"Laptop for development\",\"price\":999.99,\"stock\":10}"
```

List products:

```bash
curl "http://localhost:8080/api/products?page=0&size=20"
```

Get product by id:

```bash
curl http://localhost:8080/api/products/1
```

Update product:

```bash
curl -X PUT http://localhost:8080/api/products/1 \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Laptop Lenovo Updated\",\"description\":\"Updated product description\",\"price\":1099.99,\"stock\":8}"
```

Delete product:

```bash
curl -X DELETE http://localhost:8080/api/products/1
```

## Error handling

The API returns consistent error responses for validation, resource not found and malformed request scenarios.

Example 404 response:

```json
{
  "status": 404,
  "code": "NOT_FOUND",
  "error": "Not Found",
  "message": "Product not found with id: 999999",
  "path": "/api/products/999999",
  "timestamp": "2026-07-09T18:20:45.5822458",
  "fieldErrors": null
}
```

- Validation errors return `400` with `fieldErrors`.
- Invalid or missing request bodies return `400`.
